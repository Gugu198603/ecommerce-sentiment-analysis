## 1. 项目背景与研究动机

在电子商务时代，用户评论已成为消费者购买决策和商家优化产品服务的重要参考。本项目旨在通过自然语言处理（NLP）技术，对京东等电商平台的商品评论进行抓取和情感倾向分析（正向/负向）。

### 1.1 研究目标
1. 自动化获取特定商品的评论数据。
2. 摒弃传统的词袋模型，利用更深层次的词向量表示（Word2Vec）与预训练语言模型（BERT）进行情感分类。
3. 对比传统机器学习（SVM/逻辑回归）与深度学习模型（BERT）在情感分析上的性能差异。
4. 结合可视化手段提供业务洞察，探讨评论情感对购买决策的影响。

## 2. 数据获取与预处理

### 2.1 数据获取
使用 `Requests` 结合 `BeautifulSoup` 和 JSON 解析技术，针对京东商品评论接口（JSONP格式）进行数据采集。主要字段包括：用户评论内容、评分（1-5分）、发布时间、点赞数等。
- **爬取数量：** 5000+ 条
- **反爬策略：** 动态 User-Agent、请求随机延时。

### 2.2 数据清洗与平衡
1. **去噪与清洗：** 去除 HTML 标签、URL、特殊表情符号，仅保留中文字符和基础标点。
2. **情感标签映射：** 规定评分 $\ge 4$ 为正向情感（标签为1），评分 $\le 2$ 为负向情感（标签为0），3分为中性进行丢弃处理。
3. **分词处理：** 使用 `jieba` 中文分词库进行精准模式切词。
4. **样本平衡：** 针对可能出现的正负面样本极度不均衡情况，采用随机重采样（Resampling）策略将正负样本控制在 1:1，以避免模型出现严重偏向。

## 3. 模型构建与实验设计

本项目选用了三种具有代表性的模型进行对比实验，以体现技术的演进：

### 3.1 基于Word2Vec的传统机器学习模型
**为什么不用词袋模型(BoW)？**
词袋模型忽略了词语之间的上下文关系和语义相似性，且容易导致维度灾难和数据稀疏。本项目采用 `gensim` 训练 Word2Vec 词向量，将每个句子的词向量进行平均（Mean Pooling），得到低维稠密的句子表示。随后输入至：
1. **逻辑回归 (Logistic Regression)** 
2. **支持向量机 (SVM)** (基于 RBF 核)

### 3.2 深度学习预训练模型 (BERT)
采用 Google 开源的 `bert-base-chinese` 预训练模型。
- **模型结构：** Tokenizer 提取 `[CLS]` 向量，连接一个线性分类层。
- **参数设置：** Learning Rate = 2e-5, Batch Size = 16, Epochs = 3。
- **优化器：** AdamW。

## 4. 实验结果与对比分析

### 4.1 模型性能对比表

| 模型 | 特征表示 | Accuracy | F1-Score | 优缺点分析 |
|---|---|---|---|---|
| **逻辑回归** | Word2Vec (100维) |  |  | 速度快，解释性强，但无法捕捉深层语义。 |
| **SVM** | Word2Vec (100维) |  |  | 在非线性切分上表现较好，但对于长文本序列仍然受限。 |
| **BERT** | BERT-Base |  |  | 准确率最高，F1表现最佳，由于自注意力机制能深刻理解上下文，但训练成本较高。 |

## 5. 高级功能扩展

为了提升系统的实用性和工程化水平，本项目在基础情感分类的基础上进行了两大扩展：
1. **基于 LLM 的细粒度情感分析 (ABSA)：** 引入了大语言模型（如 GPT-3.5 或兼容模型），实现了对单条评论中“物流”、“质量”、“服务”等具体属性的精准情感倾向拆解。
2. **前后端分离架构与可视化系统：**
   - 使用 **FastAPI** 部署了支持高并发的后端预测接口，将复杂的 BERT 推理封装为 RESTful API。
   - 使用 **Streamlit** 搭建了直观友好的 Web 交互前端。用户不仅可以进行单条评论的实时打分，还能一键生成包含词云和分布图的商业大盘分析。

## 6. 结论与业务建议

1. **模型结论：** 深度学习（BERT）在电商短文本情感分析中显著优于传统机器学习（Word2Vec+SVM），它能够更好地处理“双重否定”、“反讽”等复杂语境。
2. **管理启示：** 情感负面的评论对潜在消费者的购买决策具有极大的劝退作用（从点赞数可以反映出群体认同）。建议商家设立预警机制，一旦出现针对“物流”和“质量”的长篇负面评论，客服需在2小时内介入安抚。
