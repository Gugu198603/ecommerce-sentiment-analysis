# 面试 Q&A 准备指南：电商评论情感分析项目

这部分文档专门为你应对求职面试（数据分析师/算法工程师）准备。请熟读以下核心问题。

---

## 1. 业务与项目背景

**Q: 为什么你要做这个项目？**
> **回答要点**：为了解决电商领域的实际业务痛点。用户评论是了解产品口碑和用户反馈的最直接渠道。通过自动化情感分析，能帮助商家快速定位负面舆情（如“物流慢”、“包装破损”），提升产品迭代速度。这不仅是一个NLP技术展示，更是技术赋能业务（商务智能）的完整闭环。

**Q: 你在项目中遇到了什么困难？是如何解决的？**
> **回答要点**：
> 1. **反爬虫机制**：京东评论接口有JSONP跨域和访问频率限制。我通过正则表达式提取JSON有效数据，并设置了动态延时和伪装Header来解决。
> 2. **数据极度不平衡**：电商评论往往好评占绝大多数（>90%）。如果直接训练，模型会变成只猜“好评”的废模。我使用了重采样（过采样/欠采样）技术，将正负样本强制平衡到1:1，保证了模型能真正学会分辨“差评”。

## 2. NLP 算法核心原理

**Q: 为什么你在对比实验中坚持不用“词袋模型 (Bag of Words) / TF-IDF”，而使用了 Word2Vec？**
> **回答要点**：
> 1. **维度灾难与稀疏性**：词袋模型会将特征维度拉到整个词表大小（几万维），且绝大多数是0。
> 2. **语义缺失**：词袋模型无法理解“手机”和“电话”是相似的，也完全丢失了词序信息。
> 3. **Word2Vec优势**：它是一种分布式表示，把词映射到低维（如100维）稠密向量空间中。语义相似的词在空间距离上更近，能够为SVM和LR提供更好的特征输入。

**Q: 请解释一下 BERT 的 Mask 机制（Masked Language Model, MLM）。**
> **回答要点**：
> BERT 的核心是双向 Transformer 编码器。在预训练时，它为了防止“信息泄露”（也就是词在预测自己之前就看到了自己），引入了 Mask 机制。
> 随机选择 15% 的词进行遮盖（其中 80% 替换为 `[MASK]`，10% 替换为随机词，10% 保持不变）。然后让模型根据上下文去预测这些被遮盖的词。这迫使模型真正去理解前后文的深层语义，因此在情感分析这种高度依赖上下文的任务中表现惊艳。

**Q: 你的 BERT 模型在代码里是怎么做情感分类的？**
> **回答要点**：
> 我使用了 `BertForSequenceClassification`。输入文本经过 Tokenizer 处理后，在句子开头会加上 `[CLS]` 标记。BERT 跑完后，会提取这个 `[CLS]` 对应的隐藏状态（它汇聚了整个句子的全局语义），然后接一个简单的全连接层（Linear Layer），输出 2 维（正向和负向）的 logits，最后用交叉熵损失函数（CrossEntropyLoss）进行优化。

## 3. Git 与工程化

**Q: 你的项目代码是如何管理的？能否简单说几个你常用的 Git 命令？**
> **回答要点**：
> 我使用 Git 进行版本控制，并推送到 GitHub。为了保证仓库干净，我配置了 `.gitignore` 过滤掉了 `__pycache__`、数据集文件和过大的模型权重（这些应该放云盘）。
> 常用的命令：
> - `git init`：初始化仓库
> - `git add .`：暂存修改
> - `git commit -m "feat: add crawler module"`：规范化提交代码
> - `git push origin main`：推送到远程仓库

**Q: 代码里的 `tqdm` 和 `logging` 是为了什么？**
> **回答要点**：
> 这是为了工程规范和用户体验。数据处理和模型训练都是耗时操作，`tqdm` 能提供清晰的进度条，`logging` 替代了简单的 `print`，方便后续将日志重定向到文件进行排错，展示了良好的代码编写习惯。
